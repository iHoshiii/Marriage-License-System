# Slide 3: INTRODUCTION
**WHAT IS THE PROBLEM?**  
Manual marriage license processing often suffers from slow and manual errors in encoding, retrieval of records, and a lack of real-time tracking the applicants.

**WHAT ARE THE SOLUTIONS WE THINK ARE HELPFUL/USEFUL?**  
We decided to make a modern, centralized, role-based system that automates data entry, provides instant status updates, and generates analytical reports for municipal decision-making.

# Slide 8:
EMPLOYEE  
WHATS AND HOWS:
* Employee can create a walk-in application or on-site application.
* Employee have the access to view, edit, download, give registry number, take a photo and set the status of the applicant.
* Employee can see all the pending, approved, completed and rejected applicant.

# Slide 9:
**what can admin do?**  
The ADMIN has the full control and access in the global oversight of the system. He can create, edit, delete, download, take a picture, give registry number and set the status of the applicant. Admins can also assign employees, promote them, demote them, create account for employee, etc.

**what can admin access?**  
One of our highlight feature in ADMIN role is that he/she can download the reports and analytics in the specific time range he/she want. This includes all the data of applicants oldest to latest (FILO), analytics (data based on addresses, religion, age distribution and others), and last are the Completed Applications with their registry number sorted in ascending number (FIFO).
